# THC Methodology Adoption Note

This is a suggested replacement note for implementation repositories such as `project-scaffold`.

## Short Version

```md
# THC Methodology

This project follows THC Methodology.

Canonical source:
https://github.com/Vel-Labs/thc-methodology

Local implementation notes live in:
docs/repo-truth/
```

## Longer Version

```md
# THC Methodology

This repository is an implementation surface for THC Methodology.

THC stands for:

- Truth
- Hardening
- Clarity

The canonical methodology lives at:

https://github.com/Vel-Labs/thc-methodology

This repository uses THC to reduce hidden trust in project structure, documentation, validation, and agentic coding workflows.

Local repo-truth artifacts live in:

docs/repo-truth/
```

## Reason

The methodology should be canonical in its own repository.

Implementation repositories should point to it and document how they apply it locally.
