# THC Case Study Discipline

THC should not be validated only by successful projects.

That would create survivorship bias.

The point of a THC case study is not to prove that THC guarantees project success.

It does not.

The point is to evaluate how much hidden trust a project requires from users, contributors, operators, maintainers, and agents.

## Required Case Study Types

A serious case study set should include:

| Type | Why It Matters |
|---|---|
| High-THC project that succeeded | Shows the value of low hidden trust. |
| High-THC project that failed | Prevents overclaiming. |
| Low-THC project that succeeded | Shows that low hidden trust is not the only success factor. |
| Low-THC project that failed | Shows common reliability collapse points. |
| Popular project with maintainer folklore | Tests whether openness equals clarity. |
| Small project with strong clarity | Shows that size is not required for good posture. |
| Agent-heavy project | Tests whether instructions and validation are agent-legible. |

## Questions For Every Case Study

- What did the project claim?
- What was actually true?
- Where was trust hidden?
- Which assumptions were hardened?
- Which assumptions were not hardened?
- What was clear to outsiders?
- What required private context?
- What failed?
- What survived?
- What would THC have caught earlier?
- What would THC not have caught?

## Avoid

Do not write case studies that only say:

```txt
This project followed THC and succeeded.
```

That is marketing, not methodology.

A useful case study should include uncomfortable findings.

## Counterexample Discipline

Include counterexamples.

Examples:

- A project with great docs but poor reliability.
- A project with strong tests but unclear operator behavior.
- A project with minimal docs that still succeeds because its system boundary is tiny and obvious.
- A project that scores high on THC but fails for market, funding, governance, or adoption reasons.

## Case Study Conclusion Format

End every case study with:

```md
## THC Finding

## What THC Explains

## What THC Does Not Explain

## Hidden Trust Still Present

## Practical Takeaway
```

This keeps the methodology honest.
